# Research Synthesis: How Children Learn Math (Ages 10–14)

## Compiled for Dr. Math — Class VII Content Pipeline

---

## 1. The Core Insight: Every Child's Brain Evolves Differently

**Bloom's 2 Sigma Problem (1984):** Students with one-on-one tutoring + mastery learning perform **2 standard deviations** better than classroom instruction. That's moving from 50th percentile to **98th percentile**.

**What this means for us:** The system must feel like a personal tutor that adapts to *this specific child* — not a rigid curriculum.

---

## 2. How Kids Actually Learn (Cognitive Science)

### Piaget's Transition (Ages 10–14)
Kids move from **Concrete Operational** (need physical/tangible examples) to **Formal Operations** (can handle abstract reasoning). But this transition is **gradual and uneven**.

**Practical rule:** Never start with formulas. Always start with something the child can touch, imagine, or relate to — then move to symbols.

### Vygotsky's Zone of Proximal Development (ZPD)
The magic happens when a problem is **just beyond** what the child can do alone — but achievable with a nudge.

**Practical rule:** If the kid gets 3 right in a row, make the next question harder. If they get 2 wrong, make it easier or give a hint. Don't wait for a test.

### Working Memory Limits
A 12-year-old can hold about **5–7 items** in working memory. Complex multi-step problems overload them.

**Practical rule:** Break problems into visible steps. Don't hide intermediate steps.

## 3. What the Research Says About Questions

### Productive Struggle > Immediate Answers
Letting a child struggle for 3–5 minutes before giving a hint creates **deeper learning** than giving the answer immediately (Kapur, 2008).

### Worked Examples Work
Showing a complete step-by-step solution first — then asking the child to solve a similar problem — is more effective than throwing them in the deep end (Sweller & Cooper, 1985).

### The CPA Approach (Concrete → Pictorial → Abstract)
Singapore's top-ranked math system uses this:
1. **Concrete:** "You have 3 mangoes and get 2 more"
2. **Pictorial:** Draw 3 boxes + 2 boxes
3. **Abstract:** 3 + 2 = 5

**Practical rule:** Every explanation should include a visual. Every question should suggest "draw this."

### Cultural Context Matters
Kids solve problems **faster and with less anxiety** when the context is familiar. A 2024 Ghana study showed culturally familiar problems improved scores by **0.96 standard deviations**.

**Practical rule:** Use Indian contexts — cricket, bazaar, school, festivals, auto-rickshaw, chai stall.

## 4. Adaptive Flow (What the User Asked For)

The user wants a system where:
- There is a "mark" (the kid's current ability state)
- If they run out of time or can't solve → next question adapts
- Questions customize based on how the kid's brain is evolving
- Multi-dimensional tracking

### Research-Backed Adaptive Dimensions

| Dimension | What to Track | How to Adapt |
|-----------|--------------|--------------|
| **Accuracy** | Right/wrong streak | 3 right → harder; 2 wrong → easier |
| **Speed** | Time taken per question | Slow + wrong → simpler numbers; Fast + right → complex numbers |
| **Hints used** | Did they need help? | Needs hints → more worked examples; No hints → independent problems |
| **Error type** | What kind of mistake? | Same error twice → targeted micro-lesson on that specific misconception |
| **Confidence** | Hesitation patterns | Long pause + wrong → lower difficulty; Quick wrong → overconfidence trap question |
| **Prior knowledge** | Topics already mastered | Interleave mastered topics with new ones (spaced repetition) |

### The Flow (No Rigid Levels)

```
Child answers Question N
    ↓
System evaluates: accuracy + speed + hints + error type
    ↓
System updates "mark" (multi-dimensional state)
    ↓
System selects Question N+1 from a pool:
    - Same concept, different numbers (if struggling)
    - Same concept, harder variant (if mastering)
    - Different concept, prerequisite review (if gap detected)
    - Interleaved review (if spaced repetition due)
    ↓
Child sees Question N+1
```

**No levels. Just a continuous adaptive stream.**

## 5. Feedback Timing Research

| When to Give Feedback | Why |
|----------------------|-----|
| **Immediate** for practice problems | Prevents encoding wrong methods |
| **Slightly delayed** for conceptual challenges | Encourages deeper thinking |
| **Never punitive** | Math anxiety kills learning |
