Engineering Portfolio Case Study

Dr. Math

An adaptive, AI-powered math learning platform for Indian Class VII students. Every decision is backed by peer-reviewed learning science.

🚀 Live Demo 💻 GitHub 📄 Read Case Study

The Problem

Most math apps for Indian middle-schoolers fall into two traps:

  • Rigid difficulty levels force kids into fixed "easy/medium/hard" buckets that ignore their actual thinking.
  • Generic worksheets generate endless practice without understanding why a child struggles.

The result: children get bored or anxious, and never develop the reasoning flexibility that real math learning requires.

The Product Vision

Build an adaptive math companion that reads how a child's brain is evolving — not just whether they got the answer right — and continuously selects the next question, hint, or explanation to move them into their Zone of Proximal Development.

Pedagogical Dimensions

🧮 Default (Anti-Gravity)

Clear scaffolding and progressive difficulty via Rosenshine's Principles of Instruction.

📖 Cultural Storyteller

Math embedded in Indian cultural contexts using Situated Learning theory.

👁️ Visual-First Thinker

Diagrams, number lines, and spatial reasoning based on Bruner's CPA approach.

🎯 ZPD Adaptive Tutor

Just-harder-than-comfortable problems with fading support via Vygotsky's ZPD.

Engineering Rigor

Research-First Covenant

Decompose → BFS → DFS → ADR → Code. Every architectural decision is documented and cited.

Council of Ten Consensus

Non-trivial decisions are deliberated across 10 expert personas before code is written.

Deterministic AI Rails

LLM outputs are constrained by structured generation, prompt versioning, and grounding logs.

Privacy by Default

Pseudonymous student IDs, minimal PII, encrypted state at rest, child-safety-first design.

What I Learned

  • AI is not the product; pedagogy is. The best technical demo fails if it does not serve how children actually learn.
  • Deterministic rails matter for kids. LLMs must be constrained by structured output and explicit taxonomies.
  • Data models outlive code. Separating events from derived state pays off across every future feature.
  • Research-first engineering is slower but safer. Writing ADRs before code prevented expensive pivots.
